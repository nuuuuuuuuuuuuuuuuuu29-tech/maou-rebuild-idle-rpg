# 魔王再建記

「魔王再建記」は、敗北した魔王が配下の魔物を集め、短時間の遠征を繰り返しながら失われた領地を取り戻していく、オリジナルの放置型・遠征型・魔物育成RPGです。

既存ゲームの固有名詞、文章、画像、UI、データ、バランスは使用していません。参考にするのは「放置・遠征・魔物育成・報酬収集・次の遠征へ進む」というゲーム体験だけです。

## 起動手順

```bash
npm install
npm run dev
```

ビルド確認:

```bash
npm run build
```

## 技術構成

- Vite + React + TypeScript
- 通常のCSS
- フロントエンドのみ
- セーブデータは `localStorage`
- 外部APIなし
- 画像素材なし。絵文字、テキスト、CSSで表現

## 主な機能

- 魔王名、魔王Lv、所持金、領地解放率の表示
- 魔王名の変更
- 魔物ユニットの所持、詳細表示、名前変更
- ダンジョン選択、複数ユニット選択、作戦方針、持ち込みアイテム
- 30秒から数分程度のリアルタイム遠征
- 遠征中の進捗バーとログ表示
- 遠征完了後の自動戦闘結果、報酬、MVP表示
- 魔王経験値、ユニット経験値、所持金、アイテム、魔物救出、領地解放
- 司令部での魔物雇用、アイテム購入、売却、保有上限拡張
- 魔物、アイテム、ダンジョンの図鑑
- localStorageによる保存とリロード後の復元
- 壊れたセーブデータのバックアップと初期化
- 設定画面からのリセット

## v0.2 やりこみ要素

v0.2では、既存の遠征サイクルを壊さずに、継続プレイの目標になる要素を追加しています。

### 実績

遠征回数、初勝利、ボス討伐、図鑑登録数、魔王Lv到達などで実績が解除されます。

実績は図鑑画面の「実績」欄で確認できます。遠征完了時に新しく解除された実績は、作戦ログにも追記されます。

### ボス討伐記録

ダンジョンごとにボス討伐回数、初討伐日、未討伐状態を記録します。

この記録は図鑑画面の「ボス討伐記録」欄で確認できます。既存セーブデータから読み込んだ場合は、残っている遠征記録から推定できる範囲で討伐記録を復元します。

### 図鑑報酬

魔物、アイテム、ダンジョン、総登録数が一定数に到達すると、図鑑画面で報酬を受け取れます。

報酬は一度だけ受け取り可能です。二重受け取りはできません。アイテム報酬がある場合、保有枠が足りないと受け取れないため、司令部で売却または上限拡張を行ってください。

## v0.3 ダンジョン熟練度

同じダンジョンを周回する意味を作るため、ダンジョンごとの成功クリア回数を記録します。

- 熟練度Lvは 1回、5回、10回、25回、50回の成功クリアで上がります。
- 熟練度ボーナスは控えめで、Lvごとに獲得金額 +2%、ユニット経験値 +1% です。
- 成功率や敵の強さには影響しません。
- 遠征準備画面と図鑑のダンジョン欄で、熟練度Lv、踏破回数、次Lvまでの回数、ボーナスを確認できます。
- 古いセーブデータからは、残っている遠征記録とボス討伐記録を元に可能な範囲で熟練度を復元します。

## v0.4 レアドロップ

v0.4では、遠征報酬のうち `Rare` / `Epic` / `Legendary` のアイテムをレアドロップとして扱います。

- レア判定は `src/data/items.ts` の `ItemDefinition.rarity` を使います。
- ダンジョンごとの `rewards` がレアドロップ候補になります。
- 遠征準備画面では、選択中ダンジョンの主な希少候補を確認できます。ただし具体的な抽選率は表示しません。
- 遠征完了時、実際にインベントリへ入ったRare以上アイテムだけをレアドロップとしてログ・図鑑・実績に反映します。
- インベントリ満杯で受け取れなかったアイテムは、初入手扱いにも図鑑登録扱いにもなりません。
- ダンジョン熟練度LvごとにRare以上アイテムの抽選だけが少し上がります。補正はLvごとに+0.3%、最大+1.5%です。
- 成功率、敵の強さ、通常アイテムの抽選率には影響しません。
- セーブデータの保存項目は増やしていないため、`GameState.version` は `4` のままです。

## v0.5 称号システム

v0.5では、実績、ボス討伐記録、図鑑、ダンジョン熟練度、レアドロップなどの進行状況から称号を獲得できるようになりました。

- 称号はステータス効果を持たない、表示・達成感のためのやりこみ要素です。
- 獲得済み称号は保存せず、現在の `GameState` から毎回導出します。
- 保存する新規項目は、ホーム画面に表示する `selectedTitleId` のみです。
- 図鑑画面の称号欄で、獲得済み称号を1つ選んで掲げられます。
- 未獲得称号は条件と進捗を確認できます。
- `GameState.version` は `5` になり、v4からv5へのマイグレーションを追加しています。

## v0.6 魔物特性・個性

v0.6では、魔物ごとに種族固定の特性を持つようになりました。遠征準備で特性を見ながら、どの魔物を連れていくかを少し考えられます。

- 特性は `UnitTemplate.id` から導出します。`GameUnit` には保存しません。
- セーブデータの新規保存項目は追加していないため、`GameState.version` は `5` のままです。
- 獲得済みユニットにも、起動時に自動で特性が表示されます。
- 特性効果は控えめで、部隊合計の上限があります。
- 反映される効果は、獲得金額、通常戦評価、ボス戦評価、被ダメージ、罠リスク、通常素材入手率、失敗時被害の小さな補正です。
- レアドロップ率への直接補正はありません。
- 特性は戦闘中または報酬計算中だけ使われ、ユニットの永続ステータスは書き換えません。

## セーブデータ

- localStorageキー: `maou-rebuild-state-v1`
- 現在のセーブバージョン: `5`
- `GameState.version` を使ってセーブデータを管理します。
- v0.2では、v2からv3へのマイグレーションを追加しています。
- v0.3では、v3からv4へのマイグレーションを追加しています。
- v0.5では、v4からv5へのマイグレーションを追加しています。
- v3で追加された保存項目:
  - `achievements`
  - `bossRecords`
  - `collectionRewards`
- v4で追加された保存項目:
  - `dungeonMastery`
- v5で追加された保存項目:
  - `selectedTitleId`
- 既存の `units`、`inventory`、`records`、`collection` は破壊的に変更しません。
- 古いセーブデータは起動時に自動移行され、移行前データはバックアップキーへ退避されます。
- 壊れたセーブデータを検知した場合も、可能な限りバックアップを残してから初期化します。

## ファイル構成

- `src/types/`: 型定義
- `src/data/`: 魔物、ダンジョン、アイテム、実績、図鑑報酬、称号、特性のデータ
- `src/lib/`: 遠征、戦闘、進行、実績、ダンジョン熟練度、称号、特性、保存処理
- `src/components/`: 画面UI
- `src/styles.css`: 全体スタイル

## テスト

重要なゲームロジックは Node 標準テストランナーで確認できます。

```bash
npm test
```

現在の自動テストでは、遠征開始、遠征完了、報酬と経験値の反映、実績解除、ボス討伐記録、ダンジョン熟練度、レアドロップ、称号判定、称号選択条件、魔物特性の集計と補正、図鑑報酬の受け取りと二重受け取り防止、保有枠不足時のブロック、localStorage保存/復元、古いセーブデータ移行、壊れたセーブデータの復旧を確認します。
