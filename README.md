# 魔王再建記

「魔王再建記」は、敗北した魔王となって配下の魔物を集め、短時間の遠征を繰り返しながら失われた領地を取り戻していく、完全オリジナルの放置型・遠征型・魔物育成RPGです。

既存ゲームの固有名詞、文章、画像、UI、データ、バランスは使用していません。参考にしているのは「放置・遠征・魔物育成・報酬収集・次の遠征へ進む」というゲーム体験と構造だけです。

## 現在の位置づけ

- v1.0公開候補に向けた安定確認中のフロントエンドゲームです。
- 実装済み範囲は v0.9「遠征準備の危険度・推奨Lv警告UI」までです。
- `GameState.version` は `5` です。
- localStorageの保存構造は安定しています。
- `package.json` の `version` は `1.1.0-alpha.1` です。敵キャラクター・詳細戦闘ログを含む v1.1 系の開発版メタデータとして扱います。
- v1.0.0-rc.2では、スマホ下部ナビを5項目＋メニュー方式へ変更し、遠征準備の初心者導線を補強しています。
- v1.1.0-alpha.1では、遠征結果と作戦記録で敵遭遇、味方攻撃、敵攻撃、HP変化、撃破、撤退、報酬の詳細戦闘ログを確認できます。

## ゲームの目的

1. 魔物を雇用・救出して配下を増やす。
2. ダンジョンへ遠征部隊を送り出す。
3. 戦利品、経験値、領地解放率を獲得する。
4. 魔王と魔物を育て、次のダンジョンを解放する。
5. 実績、図鑑報酬、称号、熟練度、レアドロップを集めて再建を進める。

## 基本の遊び方

1. ホーム画面で魔王名と「次にやること」を確認する。
2. 遠征準備画面で、最初はおすすめ表示の「煤けた境界村」を選ぶ。
3. 待機中の魔物を選び、作戦方針を選ぶ。
4. 危険度ラベルを見て、無理がないか確認する。
5. 遠征開始後はカウントダウンを待つ。
6. 作戦記録で結果、報酬、成長、ログを確認する。
7. 司令部で雇用・購入・売却・上限拡張を行い、次の遠征へ進む。

## 画面説明

### ホーム

- 魔王名、称号、魔王Lv、所持金、領地解放率を確認できます。
- 初回案内と「次にやること」が表示されます。
- 遠征中の場合は残り時間と進捗バーが表示されます。

### 配下一覧

- 所持している魔物の名前、種族、Lv、HP、ATK、DEF、SPD、レアリティ、状態を確認できます。
- ユニット詳細を開き、名前を変更できます。
- 種族固定の魔物特性も確認できます。

### 遠征準備

- ダンジョン、出撃ユニット、作戦方針、持ち込みアイテムを選べます。
- 遠征開始までのステップガイドで、次に選ぶべき項目と準備完了状態を確認できます。
- 初回は「煤けた境界村」「配下1体以上」「バランス重視」を基本導線として案内します。
- ダンジョン熟練度、希少候補、部隊特性ボーナスを確認できます。
- 危険度UIにより、現在の部隊と作戦で挑む危険度を `安全` / `注意` / `危険` / `無謀` の4段階で確認できます。
- 危険度は判断材料です。出撃は禁止しません。

### 作戦記録

- 遠征中ログ、遠征完了後の結果、報酬、MVP、実績解除、熟練度上昇、レアドロップを確認できます。
- ログはダンジョンや作戦方針に応じたダークファンタジー寄りの文体です。

### 司令部

- 魔物の雇用、アイテム購入、不要な魔物やアイテムの売却ができます。
- 魔王Lvに応じて、売却や保有上限拡張が解放されます。
- 詰まり防止のため、基本ユニットは再雇用できる設計です。

### 図鑑

- 発見済みの魔物、アイテム、ダンジョンを確認できます。
- 未発見のものは「？？？」で表示されます。
- 実績、図鑑報酬、ボス討伐記録、ダンジョン熟練度、称号も確認できます。

## 主な機能

### 遠征

- 30秒から数分程度のリアルタイム遠征です。
- 遠征中に画面を閉じても、再起動時に経過時間から完了処理を行います。
- 作戦方針は `バランス重視` / `安全重視` / `強行突破` / `戦利品重視` です。

### 魔物育成

- 魔物は戦闘で経験値を得てレベルアップします。
- HPが0になると戦闘不能になりますが、MVP段階では完全ロストはありません。
- 種族ごとにステータス傾向と固定特性があります。

### 実績

- 遠征回数、初勝利、ボス討伐、図鑑登録数、魔王Lv到達などで解除されます。
- 遠征完了時に新しく解除された実績は作戦ログにも表示されます。

### 図鑑報酬

- 魔物、アイテム、ダンジョン、総登録数が一定数に到達すると報酬を受け取れます。
- 報酬は一度だけ受け取り可能です。
- アイテム枠が足りない場合は受け取りがブロックされます。

### ボス討伐記録

- ダンジョンごとにボス討伐回数、初討伐日、最終討伐日を記録します。
- 古いセーブデータでは、残っている遠征記録から推定できる範囲で復元します。

### ダンジョン熟練度

- ダンジョンごとの成功クリア回数を記録します。
- 熟練度Lvは 1回、5回、10回、25回、50回の成功クリアで上がります。
- 熟練度ボーナスは控えめで、Lvごとに獲得金額 +2%、ユニット経験値 +1% です。
- 成功率や敵の強さには影響しません。

### レアドロップ

- `Rare` / `Epic` / `Legendary` のアイテムをレアドロップとして扱います。
- 実際にインベントリへ入ったRare以上アイテムだけが、図鑑・実績・ログに反映されます。
- インベントリ満杯で受け取れなかったアイテムは、初入手扱いにも図鑑登録扱いにもなりません。
- ダンジョン熟練度によりRare以上アイテムの抽選だけが少し上がります。補正はLvごとに +0.3%、最大 +1.5% です。

### 称号

- 実績、ボス討伐、図鑑、熟練度、レアドロップなどの進行状況から称号を獲得できます。
- 称号は表示・達成感のための要素で、ステータス効果はありません。
- 保存するのは選択中称号の `selectedTitleId` のみです。獲得済み称号一覧は現在のGameStateから導出します。

### 魔物特性

- 各魔物は `UnitTemplate.id` から導出される種族固定特性を持ちます。
- `GameUnit` には特性IDを保存しないため、既存セーブにも自動で反映されます。
- 効果は控えめで、獲得金額、通常戦評価、ボス戦評価、被ダメージ、罠リスク、通常素材入手率、失敗時被害などに小さく影響します。
- レアドロップ率への直接補正はありません。

### 危険度UI

- 遠征準備画面で、選択中の部隊、ダンジョン、作戦方針から危険度を導出します。
- 成功率の具体的な%は表示しません。
- 出撃は禁止せず、短い理由とおすすめ行動を表示します。
- ダンジョン難易度、敵ステータス、報酬、作戦補正は変更していません。

### スマホUIと初心者導線

- スマホ幅では下部ナビゲーションを `ホーム` / `配下` / `遠征` / `記録` / `メニュー` の5項目に抑え、本文領域を確保します。
- `司令部` / `図鑑` / `設定` はスマホ幅ではメニュー内から移動できます。
- 遠征準備画面では、現在の選択状態に応じて「次はこちら」マーカーを表示します。
- 遠征開始できない場合は、開始ボタン付近に不足条件を表示します。
- 出撃可能になると「準備完了」として遠征開始ボタンを強調します。
- ガイド表示は画面上の導線補助のみで、セーブデータや出撃条件は変更しません。

## セーブ仕様

- 保存先: ブラウザの `localStorage`
- localStorageキー: `maou-rebuild-state-v1`
- 現在のセーブバージョン: `5`
- v3で追加された保存項目: `achievements`, `bossRecords`, `collectionRewards`
- v4で追加された保存項目: `dungeonMastery`
- v5で追加された保存項目: `selectedTitleId`
- 古いセーブデータは起動時に自動移行されます。
- 移行前データはバックアップキーへ退避されます。
- 壊れたセーブデータを検知した場合も、可能な限りバックアップを残してから初期化します。
- 設定画面からリセットできます。リセット前には確認ダイアログを表示します。

## 起動方法

初回のみ依存関係をインストールします。

```bash
npm install
```

開発サーバーを起動します。

```bash
npm run dev
```

ブラウザで表示されたURLを開いてください。通常は `http://localhost:5173` です。

### Windows PowerShellの場合

この環境では `npm.ps1` の実行ポリシーで止まる場合があります。その場合は `npm` ではなく `npm.cmd` を使います。

```bash
npm.cmd install
npm.cmd run dev
```

## ビルド方法

```bash
npm run build
```

Windows PowerShellでは次を使います。

```bash
npm.cmd run build
```

## テスト方法

重要なゲームロジックは Node 標準テストランナーで確認できます。

```bash
npm test
```

Windows PowerShellでは次を使います。

```bash
npm.cmd test
```

現在の自動テストでは、遠征開始、遠征完了、報酬と経験値の反映、実績解除、ボス討伐記録、ダンジョン熟練度、レアドロップ、称号判定、称号選択条件、魔物特性、危険度判定、図鑑報酬、localStorage保存/復元、古いセーブデータ移行、壊れたセーブデータ復旧を確認します。

## バランスシミュレーション

序盤から中盤の成功率、報酬、経験値、Rare以上ドロップ率、戦闘不能率などを観測できます。ゲーム本体のlocalStorageは読み書きしません。

```bash
npm run sim:balance
```

Windows PowerShellでは次を使います。

```bash
npm.cmd run sim:balance
```

代表的な確認コマンド:

```bash
npm.cmd run sim:balance -- --trials 1000 --seed 12345
npm.cmd run sim:balance -- --json
```

詳しくは [docs/balance-simulation.md](docs/balance-simulation.md) を参照してください。

## ファイル構成

- `src/types/`: 型定義
- `src/data/`: 魔物、ダンジョン、アイテム、実績、図鑑報酬、称号、特性のデータ
- `src/lib/`: 遠征、戦闘、進行、実績、熟練度、レアドロップ、称号、特性、危険度、保存処理
- `src/components/`: 画面UI
- `src/styles.css`: 全体スタイル
- `tests/`: ゲームロジック、保存、シミュレーションの自動テスト
- `docs/`: QA、リリース、シミュレーション関連ドキュメント

## 既知の制限

- フロントエンドのみのため、セーブデータは同じブラウザのlocalStorageに保存されます。
- ブラウザやサイトデータを削除すると、通常のセーブデータも消えます。
- 画像素材は使わず、絵文字、テキスト、CSSで表現しています。
- E2Eテストはまだ導入していません。公開前は手動QAチェックリストで確認してください。
- 本格的な長時間放置、複数部隊同時遠征、装備、合成、転生は未実装です。
