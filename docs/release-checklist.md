# v1.0公開前チェックリスト

このチェックリストは、v1.0公開候補として最低限確認する項目です。コード変更やバランス調整をする前に、まず現在の状態が安定しているかを確認してください。

## 自動確認

- [ ] `git status --short` で意図しない変更がない
- [ ] `npm.cmd test` が成功する
- [ ] `npm.cmd run build` が成功する
- [ ] `npm.cmd run sim:balance -- --trials 1000 --seed 12345` が成功する
- [ ] `node_modules/`, `dist/`, `.tmp-tests/` がGit管理対象に入っていない

## 起動確認

- [ ] `npm.cmd run dev` で開発サーバーが起動する
- [ ] ブラウザでホーム画面が表示される
- [ ] console error が出ていない

## 初回プレイ10分確認

- [ ] 初回案内が表示される
- [ ] 「次にやること」が理解できる
- [ ] 最初のおすすめダンジョンが分かる
- [ ] 遠征準備まで迷わず進める
- [ ] 初回プレイ10分で遠征を3回程度試せる

## 遠征サイクル確認

- [ ] 遠征開始ができる
- [ ] 遠征中の残り時間と進捗バーが表示される
- [ ] 遠征完了後に作戦記録へ結果が出る
- [ ] 報酬、経験値、所持金、アイテムが反映される
- [ ] 魔物の経験値とレベルアップが反映される
- [ ] 遠征中にリロードしても、完了時刻後に結果処理される

## やりこみ要素確認

- [ ] 実績が解除され、図鑑画面に表示される
- [ ] 図鑑報酬を受け取れる
- [ ] 図鑑報酬を二重受け取りできない
- [ ] アイテム枠不足時、図鑑報酬の受け取りがブロックされる
- [ ] ボス討伐記録が更新される
- [ ] ダンジョン熟練度が成功時だけ増える
- [ ] レアドロップ入手時にログと図鑑へ反映される
- [ ] 称号を選択できる
- [ ] リロード後も選択中称号が復元される
- [ ] 魔物特性が配下一覧と遠征準備で確認できる

## 危険度UI確認

- [ ] 遠征準備で危険度ラベルが表示される
- [ ] ダンジョン変更で危険度が更新される
- [ ] ユニット選択変更で危険度が更新される
- [ ] 作戦変更で危険度が更新される
- [ ] 初期部隊で高難度ダンジョンを選ぶと `危険` または `無謀` になる
- [ ] `無謀` でも出撃ボタンは禁止されない

- [ ] 375px幅で図鑑、実績、称号が大きく崩れない

## セーブとリセット確認

- [ ] 魔王名変更が保存される
